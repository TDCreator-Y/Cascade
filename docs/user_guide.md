# 📖 Cascade 用户使用指南 (User Guide)

欢迎使用 Cascade Engine！本指南将带你从零开始，配置并使用这款强大的双重级联网络代理工具。

## ⚙️ 第一步：前置准备 (Prerequisites)

Cascade 是一款**双重级联代理**工具，这意味着它的海外网络穿透建立在你的“第一跳”节点之上。
在使用 Cascade 之前，请确保：
1. 你已经在本地运行了一款基础的 VPN / 翻墙软件（如 Clash, V2Ray 等）。
2. 明确该软件在本地暴露的 SOCKS5 代理端口（例如常见的 `7890` 或 `7897`）。

## 🚀 第二步：基础使用与配置

打开 Cascade 客户端，你将看到极简的暗黑配置面板。请按照以下步骤填写：

1. **本地 VPN 端口**: 填入你第一跳代理的本地端口（例如 `7897`）。
2. **远程 ISP IP 与端口**: 填入你要级联到的最终海外 ISP 纯净节点的 IP 和端口（例如 `64.51.26.139` 和 `443`）。
3. **ISP 账号与密码**: 填入远程 ISP 节点分配给你的鉴权账密。
4. **保存配置**: 点击右上角的 `保存配置`，信息将安全地持久化到本地 `config.json` 中，下次打开无需重新填写。
5. **一键启动**: 点击白色的 `启动 Cascade` 按钮。
   - 呼吸灯变为**绿色**。
   - Cascade 会立即修改 Windows 注册表并刷新网络缓存。
   - **恭喜！你现在的系统全局网络已经被 Cascade 完美接管。**

## 🛠️ 第三步：高级玩法 (CLI 独立接管)

作为开发者，全局环境变量（如给系统设置 `HTTP_PROXY` 变量）往往会带来灾难性的后果（导致很多本地脚本、Docker 镜像下载、内网测试服务出现异常）。

Cascade 提供了一种更优雅的解决方案：**CLI 工具精确注入**。

在主面板的下方，你会看到两个开关：
- **Git 代理**
- **NPM 代理**

### 如何使用？
- 只需轻轻点击开关，无需重启终端。
- 开启后，Cascade 会在底层自动执行 `git config` 和 `npm config` 指令，让 Git clone 和 NPM install 享受纯净海外网络。
- 停止 Cascade 引擎时，**所有配置会被自动还原**，绝不残留垃圾配置污染你的开发环境！

---

*如果在启动中遇到红色呼吸灯或报错，请检查你的第一跳本地 VPN 是否正常运行，以及远程 ISP 账密是否正确。*
